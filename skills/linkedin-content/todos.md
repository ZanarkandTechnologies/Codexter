# LinkedIn Content Todos

Use this as the ordered checklist whenever `linkedin-content` is active.

- [ ] Classify the artifact: text-only post, thought-leadership post, founder story, B2B announcement, hiring post, carousel outline, comment, or cross-post variant.
- [ ] State the audience, professional context, voice, goal, format, asset need, and handoff path before drafting.
- [ ] Use [research:competitor](../research/SKILL.md#researchcompetitor) or [research:parity](../research/SKILL.md#researchparity) when examples, brand/source material, platform norms, current model behavior, peer posts, category norms, or platform expectations should guide the angle.
- [ ] Use [plan](../plan/SKILL.md) when choosing angle, post format, hook, visual route, cross-post route, or scope cut.
- [ ] Load the shared [image/social production workflow](../image-generation/references/domain-production.md).
- [ ] Load the [upstream LinkedIn guide](references/upstream.md) when hook formulas, post anatomy, formatting rules, or engagement patterns matter.
- [ ] Draft the content, hook, CTA, and any visual or carousel plan before generating assets.
- [ ] Route carousel work through [social-media-carousel](../social-media-carousel/SKILL.md).
- [ ] Route still visuals through `imagegen` or [image-generation](../image-generation/SKILL.md).
- [ ] Route cross-platform campaign work through [ai-social-media-content](../ai-social-media-content/SKILL.md).
- [ ] Save drafts, prompts, inputs, generated files, and notes inside the workspace when external generation is involved.
- [ ] Confirm external compute, spend, uploads, or API usage is explicitly acceptable before running model or `belt` jobs.
- [ ] Do not publish, comment, message, or cross-post unless the user explicitly asks for that action.
- [ ] Follow the [execute](../execute/SKILL.md) proof and writeback loop before claiming professional voice, platform fit, or final quality.
