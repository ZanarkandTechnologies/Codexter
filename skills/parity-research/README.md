# Parity Research

## Purpose

Ground parity-driven product or feature decisions in comparable products,
official docs, standards, and open-source repos before narrowing the work into
`gap-analysis`, `functional-ui`, or `impl-plan`.

## Public API / Entrypoints

- `SKILL.md`: main workflow contract
- `README.md`: module summary and test checklist
- `AGENTS.md`: maintenance notes

## Minimal Example

1. Define the capability and the parity lens.
2. Capture the local baseline so the comparison stays anchored.
3. Inspect `2-5` grounded comparables across products, docs, standards, or
   repos.
4. Extract the common surfaces credible sources share.
5. Write `Repo delta` plus one recommended now/later parity boundary.
6. Hand the result to `gap-analysis`, `functional-ui`, or `impl-plan`.

## How to Test

- Confirm `SKILL.md` distinguishes parity research from `gap-analysis`.
- Confirm the workflow says broad search first, deep dive second.
- Confirm the output includes `Comparable implementations`, `Common surfaces`,
  and `Repo delta`.
