# Gotchas

- Do not run a vague prompt like "make it better"; convert it into a phase,
  owned files, and acceptance criteria.
- Do not let spec/research phases read large QA artifacts, screenshots, videos,
  frame folders, or previous session logs unless explicitly needed.
- Do not treat Pi/Kimi output as final completion. Codexter integrates and
  verifies.
- Do not mount every skill by default. Add media or framework skills only when
  the phase needs them.
- Do not allow external CLI runs to push, deploy, publish, spend unbounded
  credits, or mutate unrelated repo files.
