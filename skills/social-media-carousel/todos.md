# Social Media Carousel Todos

Use this as the ordered checklist whenever `social-media-carousel` is active.

- [ ] Classify the carousel: educational, product, announcement, thought-leadership, case-study, infographic, campaign, or recap.
- [ ] State the platform, aspect ratio, slide count, audience, hook, CTA, visual style, and final handoff path before making slides.
- [ ] Use [research:competitor](../research/SKILL.md#researchcompetitor) or [research:parity](../research/SKILL.md#researchparity) when examples, platform specs, source assets, current model behavior, peer carousels, platform norms, or swipe patterns should guide the plan.
- [ ] Use [plan](../plan/SKILL.md) when choosing carousel angle, slide structure, visual route, asset route, or scope cut.
- [ ] Load the shared [image/social production workflow](../image-generation/references/domain-production.md).
- [ ] Load the [upstream carousel guide](references/upstream.md) when platform specs, slide structure, layout rules, or examples matter.
- [ ] Draft the slide sequence before generating final visuals.
- [ ] Route static slide visuals through `imagegen` or [image-generation](../image-generation/SKILL.md).
- [ ] Route precise typography, HTML-rendered slides, or campaign pages through project code or [frontend-craft](../frontend-craft/SKILL.md).
- [ ] Save the outline, slide copy, prompts, inputs, result JSON, final image paths, and notes inside the workspace.
- [ ] Confirm external compute, spend, uploads, or API usage is explicitly acceptable before running model or `belt` jobs.
- [ ] Do not publish, post, or schedule the carousel unless the user explicitly asks for that action.
- [ ] Follow the [execute](../execute/SKILL.md) proof and writeback loop before claiming slide hierarchy, platform fit, or final quality.
