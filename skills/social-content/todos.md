# Social Content Todos

Use this as the ordered checklist whenever `social-content` or one of its
method addresses is active.

- [ ] Select one primary method: `social-content:cross-platform`,
  `social-content:carousel`, `social-content:linkedin`, or
  `social-content:twitter-thread`. Add a supporting method only when the
  artifact genuinely spans formats.
- [ ] Classify the requested artifact: post, thread, carousel, short video,
  feed image, thumbnail, caption set, hashtag set, UGC concept, content
  calendar, comment, or cross-platform campaign bundle.
- [ ] State the platform set, audience, format, output count, dimensions,
  voice/tone, CTA, asset needs, and handoff path before producing content.
- [ ] Use [research:competitor](../research/SKILL.md#researchcompetitor) or
  [research:parity](../research/SKILL.md#researchparity) when examples,
  platform specs, brand/source assets, current model behavior, peer posts,
  category norms, swipe patterns, or campaign expectations should guide scope.
- [ ] Use [plan](../plan/SKILL.md) when choosing platforms, formats, hooks,
  content angle, visual route, asset route, cross-post route, or scope cut.
- [ ] Load the shared [image/social production workflow](../image-generation/references/domain-production.md).
- [ ] For `social-content:cross-platform`, load the
  [cross-platform social guide](references/upstream-social.md) when platform
  formats, workflows, or examples matter.
- [ ] For `social-content:carousel`, load the
  [carousel guide](references/upstream-carousel.md) when slide structure,
  layout rules, swipe psychology, platform dimensions, or examples matter.
- [ ] For `social-content:linkedin`, load the
  [LinkedIn guide](references/upstream-linkedin.md) when hook formulas, post
  anatomy, formatting rules, or engagement patterns matter.
- [ ] For `social-content:twitter-thread`, load the
  [Twitter/X guide](references/upstream-twitter.md) when thread structures,
  hooks, character limits, formatting, or examples matter.
- [ ] Draft copy, hooks, CTAs, thread outline, slide sequence, prompts, or asset
  plans before generation or rendering.
- [ ] Route still visuals through `imagegen` or
  [image-generation](../image-generation/SKILL.md).
- [ ] Route video clips through [video-generation](../video-generation/SKILL.md),
  or through [remotion](../remotion/SKILL.md) and
  [remotion-render](../remotion-render/SKILL.md) for deterministic
  code-rendered video.
- [ ] Route precise typography, HTML-rendered slides, campaign pages, or web
  integration through project code or [frontend-craft](../frontend-craft/SKILL.md).
- [ ] Save drafts, outlines, slide copy, prompts, inputs, result JSON, generated
  files, final asset paths, and notes inside the workspace when external
  generation is involved.
- [ ] Confirm external compute, spend, uploads, or API usage is explicitly
  acceptable before running model or `belt` jobs.
- [ ] Do not publish, post, reply, comment, DM, schedule, or cross-post unless
  the user explicitly asks for that action.
- [ ] Follow the [execute](../execute/SKILL.md) proof and writeback loop before
  claiming platform fit, campaign readiness, professional voice, slide
  hierarchy, thread quality, or final quality.
