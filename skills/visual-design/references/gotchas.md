# Gotchas

- Do not use this skill when UX/workflow is still broken; route to `functional-ui`.
- Do not default to trendy palettes, generic cards, or gradient text.
- Do not import a new visual system when the existing one only needs hierarchy fixes.
- Do not make repeated-use product tools look like marketing pages.
- Do not use two-column layouts just because content has two categories. If the
  user reads it in order, stack it. Columns are for simultaneous comparison,
  inspector/file-viewer workspaces, split editors, or dashboard scan paths.
- Do not put a bordered div inside another bordered div unless the inner block
  is a real card, selectable item, nested tool, modal, or file/workspace pane.
  For ordinary subsections, use spacing, simple headings, dividers, or a
  low-contrast band instead.
- Do not render dense required-data/checklist content as row after row when a
  compact grid of small items would scan better.
- Do not forget responsive text fit, focus states, contrast, and reduced motion.
