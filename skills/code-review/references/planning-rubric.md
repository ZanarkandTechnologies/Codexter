# Planning Rubric

Use when reviewing a plan, technical implementation brief, or execution design.

Required dimensions:

- problem fit
- data / control flow clarity
- touched-area realism
- risk awareness
- proof strategy

Questions:

- Does the plan solve the requested problem instead of a nearby one?
- Is the data flow concrete enough to implement without guessing?
- Are the touched areas believable and scoped?
- Are the main risks and weak assumptions explicit?
- Would an implementer know how success will be proved?

Default threshold:

- any unclear proof strategy should force `revise`
- any obviously missing flow should force `revise`
