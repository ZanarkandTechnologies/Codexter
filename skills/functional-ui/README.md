# Functional UI

## Purpose

Guide agents to design product workflows from user stories and adjacent-product patterns before visual styling starts.

## Public API / Entrypoints

- `SKILL.md`: main workflow contract
- `AGENTS.md`: maintenance rules

## Minimal Example

1. Identify the user and top stories.
2. Compare similar apps and their proven workflows.
3. Produce 3 viable UI options.
4. Recommend one workflow.
5. Hand the result to `frontend-design`.

## How to Test

- Confirm the output starts with user stories.
- Confirm the artifact compares similar apps.
- Confirm one workflow is explicitly recommended.
