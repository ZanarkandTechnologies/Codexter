# Explainer Video Guide Todos

Use this as the ordered checklist whenever `explainer-video-guide` is active.

- [ ] Classify the explainer: how-it-works, onboarding, tutorial, product demo, walkthrough, feature spotlight, problem/solution, or CTA sequence.
- [ ] State the audience, concept, structure, duration, aspect ratio, voice/narration need, output format, and final handoff path before producing assets.
- [ ] Use [research:competitor](../research/SKILL.md#researchcompetitor) or [research:parity](../research/SKILL.md#researchparity) when examples, product/source assets, official behavior, current model behavior, peer explainers, product-demo norms, or platform expectations should guide the brief.
- [ ] Use [plan](../plan/SKILL.md) when choosing structure, narrative angle, visual carrier, model family, or production route.
- [ ] Load the shared [domain video production workflow](../video-generation/references/domain-production.md).
- [ ] Load [explainer video prompting](references/prompting.md) before writing prompts for problem scenes, feature demos, UI walkthroughs, motion graphics, or CTA shots.
- [ ] Draft script beats, scene list, asset list, and proof plan before generation or rendering.
- [ ] Route model-native clips through [video-generation](../video-generation/SKILL.md).
- [ ] Route still assets, posters, or references through `imagegen` or [image-generation](../image-generation/SKILL.md).
- [ ] Route deterministic motion graphics or code-rendered assembly through [remotion](../remotion/SKILL.md) and [remotion-render](../remotion-render/SKILL.md).
- [ ] Save scripts, prompts, inputs, result JSON, generated files, and notes inside the workspace when external generation is involved.
- [ ] Confirm external compute, spend, uploads, or API usage is explicitly acceptable before running model or `belt` jobs.
- [ ] Do not publish, post, or upload the video unless the user explicitly asks for that action.
- [ ] Follow the [execute](../execute/SKILL.md) proof and writeback loop before claiming narrative, platform-fit, render, or final production quality.
