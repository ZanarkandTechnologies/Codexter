# Twitter/X Thread Creation Todos

Use this as the ordered checklist whenever `twitter-thread-creation` is active.

- [ ] Classify the artifact: single post, hook tweet, thread, reply chain, media-supported thread, quote-post draft, or cross-platform variant.
- [ ] State the audience, goal, hook, thread length, media needs, voice, CTA, and handoff path before drafting.
- [ ] Use [research:competitor](../research/SKILL.md#researchcompetitor) or [research:parity](../research/SKILL.md#researchparity) when examples, source material, platform norms, current model behavior, peer threads, category norms, or platform expectations should guide the structure.
- [ ] Use [plan](../plan/SKILL.md) when choosing hook, thread structure, media route, cross-post route, or scope cut.
- [ ] Load the shared [image/social production workflow](../image-generation/references/domain-production.md).
- [ ] Load the [upstream Twitter/X guide](references/upstream.md) when thread structures, hooks, character limits, formatting, or examples matter.
- [ ] Draft the thread outline and standalone post logic before generating visuals.
- [ ] Route visuals through `imagegen` or [image-generation](../image-generation/SKILL.md).
- [ ] Route cross-platform campaign work through [ai-social-media-content](../ai-social-media-content/SKILL.md).
- [ ] Route web/campaign integration through [frontend-craft](../frontend-craft/SKILL.md).
- [ ] Save drafts, prompts, inputs, generated files, and notes inside the workspace when external generation is involved.
- [ ] Confirm external compute, spend, uploads, or API usage is explicitly acceptable before running model or `belt` jobs.
- [ ] Do not publish, reply, DM, schedule, or cross-post unless the user explicitly asks for that action.
- [ ] Follow the [execute](../execute/SKILL.md) proof and writeback loop before claiming thread quality, platform fit, or final readiness.
