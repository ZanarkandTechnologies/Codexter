# Visual Design

## Purpose

Define frontend look, taste, hierarchy, and visual-system direction once the functional UI shape is known.

## Public API / Entrypoints

- `SKILL.md`: main visual-design workflow
- `references/*`: register, numeric taste dials, design brief, and critique guidance

## Minimal Example

1. Confirm `functional-ui` has settled the workflow.
2. Classify product vs brand register.
3. Set scene sentence, numeric taste dials, and visual system.
4. Create or update a durable design brief for substantial UI work.
5. Hand constraints to `frontend-craft`.

## How to Test

- Confirm the skill routes broken workflows back to `functional-ui`.
- Confirm the output gives implementation-ready visual constraints.
- Run `python3 skills/skill-creator/scripts/quick_validate.py skills/visual-design`.
