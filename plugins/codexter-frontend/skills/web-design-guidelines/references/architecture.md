# Architecture

`web-design-guidelines` is a thin standards-audit skill. It fetches the latest
upstream Web Interface Guidelines command and applies it to selected UI source
files.

It does not own product UX, visual taste, browser proof, or anchored scoring:

- `functional-ui` owns workflow and interaction shape.
- `visual-design` owns look, taste, and visual system.
- `frontend-craft` owns implementation orchestration.
- `visual-qa` owns rendered UI judgment.
- `review/references/frontend-guidelines.md` owns conversion from findings to a
  `1.0`-to-`5.0` review score.
