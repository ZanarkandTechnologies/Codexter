# McKinsey Issue-Tree Lens

Use this lens when the problem needs a sharper decomposition into drivers,
workstreams, or root causes before choosing a design or execution path.

## Core Rule

Break the problem into branches that are as close as possible to:

- **mutually exclusive**
- **collectively exhaustive**

The goal is not perfection. The goal is to avoid overlapping buckets and blind
spots.

## Core Questions

1. **What is the top-level question?**
   - What are we really trying to solve or decide?
2. **What are the main branches?**
   - Which 2-5 top-level drivers or workstreams explain the problem?
3. **What are the sub-branches?**
   - For each branch, what sub-issues or sub-workstreams matter?
4. **What evidence would prove each branch matters?**
   - Which facts or examples would confirm or rule it out?
5. **Which branch should be tackled first?**
   - Where is the highest leverage or largest uncertainty?

## Decomposition Output

- `Top question`
- `Main branches`
- `Sub-branches`
- `Evidence needed`
- `Recommended first branch`

## Good Fit

- root-cause analysis
- complex strategy framing
- ambiguous system decomposition
- breaking a broad product problem into clearer workstreams

## Warning Sign

If branches overlap heavily or are a mix of:

- users
- components
- solutions
- symptoms

without a single organizing logic, the issue tree is not ready.
