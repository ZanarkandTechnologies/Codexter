# Workflows

## App UI Implementation

1. Confirm `functional-ui` has settled the workflow or skip with reason.
2. Confirm `visual-design` or the existing design system has settled the look.
3. Use shadcn, AI Elements, registries, and theming references as needed.
4. Implement with repo patterns.
5. Verify with tests and UI proof when visible behavior changes.

## Component Sourcing

1. Check existing components first.
2. Search shadcn/registry sources only when reuse helps.
3. Customize imported components to match the visual brief.
4. Avoid default unstyled registry output.
