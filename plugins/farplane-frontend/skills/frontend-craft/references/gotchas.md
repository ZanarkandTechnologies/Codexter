# Gotchas

- Do not let `frontend-craft` bypass the granular skills when the request clearly needs them.
- Do not treat `frontend-design` as the taste brain; it is now implementation support.
- Do not make product tools cinematic unless the operator asked for a landing/brand surface.
- Do not assume external asset tools are installed.
- Do not claim GSAP correctness from stale local examples; use official GreenSock skills or docs.
- Do not finish without proof for visible UI changes.
- Do not import frontend packages before checking `package.json`.
- Do not skip Tailwind major-version checks when writing Tailwind config or CSS.
- Do not let registry components remain in default skin when a visual brief or existing design system exists.
- Do not apply an entire shadcn/tweakcn preset when only theme or font should change.
