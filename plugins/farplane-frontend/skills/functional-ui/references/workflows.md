# Workflows

## Broken Existing UI

1. Identify primary user and action.
2. Diagnose the current UI failure.
3. Inspect comparable patterns.
4. Compare three redesign paths.
5. Recommend one path.
6. Define states and interaction rules.
7. Hand off to `visual-design` or `frontend-craft`.

## New Screen Or Flow

1. Extract user stories and constraints.
2. List states and data ranges.
3. Inspect comparable apps.
4. Compare three interaction models.
5. Recommend one model.
6. Package implementation handoff.
