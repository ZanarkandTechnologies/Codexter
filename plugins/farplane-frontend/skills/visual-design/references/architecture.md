# Architecture

`visual-design` owns visual direction after the workflow is known.

Inputs:

- `functional-ui` handoff,
- existing design system,
- brand/product context,
- user taste references or anti-references.

Outputs:

- register,
- scene sentence,
- taste dials,
- visual system,
- anti-slop constraints,
- implementation handoff.

It feeds `frontend-craft` and `frontend-design` references.
