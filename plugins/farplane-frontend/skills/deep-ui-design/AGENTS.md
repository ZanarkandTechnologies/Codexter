# `skills/deep-ui-design/AGENTS.md`

Rules for the `deep-ui-design` skill surface.

## Purpose

`skills/deep-ui-design/` owns the ambiguity-reduction contract that sharpens a
user's visual taste before `visual-design`, `landing-page`, or UI-focused
planning.

## Keep

- one-question-at-a-time taste extraction
- explicit ambiguity scoring and readiness gates
- strong separation from workflow/IA planning
- reusable `Taste Brief` outputs on live Farplane surfaces
- anti-slop and anti-reference rules before handoff

## Do Not

- drift into workflow decomposition owned by `functional-ui`
- collapse into generic mood-board prose without reusable rules
- bypass anti-reference or anti-slop clarification
- implement UI directly from inside the interview mode
