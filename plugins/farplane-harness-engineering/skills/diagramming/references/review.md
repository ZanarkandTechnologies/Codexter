# Diagramming Review

Use this checklist before handing off a diagram pack.

## Must Pass

- Can the reader understand the main change from the first diagram?
- Is there only one top-level diagram by default?
- If there is a second diagram, does it answer a distinct unresolved question?
- Does the legend make `keep / change / add / remove` obvious?
- Are inline signatures short and actually helpful?
- Would the output still make sense if the colors were lost?

## Ask If Relevant

- Is a data-flow trace better than more prose here?
- Is a zoom-in actually needed, or is the top-level map enough?
- Are we showing system shape, not just relabeling text as Mermaid?

## Fail If

- the diagram is decorative
- separate before/after diagrams are used without a strong reason
- the nodes contain paragraphs
- the reader still has to read the prose first to understand the diagram
