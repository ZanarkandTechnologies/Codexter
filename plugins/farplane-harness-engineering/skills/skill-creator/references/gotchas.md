# Skill Creator Gotchas

- Do not create a skill for generic knowledge the model already has.
- Do not hide the default execution path in references.
- Do not create empty reference files just to look structured.
- Do not make `SKILL.md` an encyclopedia.
- Do not omit judgement questions when a skill depends on metric, scope, or
  adoption calls.
- Do not add scripts without running them.
- Do not duplicate the same rule in several references.
