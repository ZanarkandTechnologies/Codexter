# Architecture

`landing-page` owns one-page story and section structure.

Inputs:

- offer,
- audience,
- product/place/person/object details,
- proof assets,
- taste brief or visual constraints.

Outputs:

- story arc,
- section map,
- visual scenes,
- asset plan,
- motion plan,
- QA plan,
- implementation handoff to `frontend-craft`.

It uses `visual-design` for visual system and official GreenSock skills or docs for GSAP details.
